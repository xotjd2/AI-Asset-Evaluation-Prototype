# AI 기반 금융상품 평가 프로젝트 기획서

## 1. 프로젝트 개요

이 프로젝트는 주식, 채권, 프로젝트 투자안을 하나의 화면에서 평가하고 결과를 저장할 수 있는 금융상품 평가 프로토타입이다. 단순 수익률 계산이 아니라 상품 유형별 위험지표를 함께 제시해 비교와 판단을 돕는 것을 목표로 한다.

## 2. 기획 배경

- 자산 유형마다 계산 방식이 달라 수작업 검토가 반복된다.
- 수익성만 보면 변동성, 금리 민감도, 부도확률 같은 핵심 위험이 누락되기 쉽다.
- 평가 결과를 저장하고 최근 이력을 다시 확인할 수 있는 도구가 필요하다.

## 3. 목표

- 주식, 채권, 프로젝트 평가를 하나의 대시보드에서 제공한다.
- 상품 유형에 맞는 위험지표를 자동 계산한다.
- 계산 결과와 설명 문구를 함께 제공한다.
- 최근 평가 결과를 저장하고 다시 조회할 수 있게 한다.
- 배포 가능한 형태의 MVP를 빠르게 운영한다.

## 4. 대상 사용자

- 금융상품 초안 검토가 필요한 사용자
- 투자안 비교가 필요한 기획 및 운영 담당자
- 시연용 프로토타입이 필요한 프로젝트 팀

## 5. 핵심 기능

### 5.1 주식 평가

- 기대수익률과 변동성 기반 평가금액 계산
- VaR 95% 제공
- 종합점수와 위험등급 산출
- 세부 설명과 민감도 분석 제공

### 5.2 채권 평가

- 할인 현금흐름 기반 현재가치 계산
- 듀레이션, 컨벡서티 제공
- 금리 충격 시 손실 추정
- 종합점수와 위험등급 산출

### 5.3 프로젝트 평가

- NPV, IRR, 회수기간 계산
- 부도확률을 반영한 신용위험 추정
- 연도별 현금흐름 입력 지원
- 종합점수와 위험등급 산출

### 5.4 평가 이력 관리

- 결과 저장 버튼으로 DB 적재
- 최근 저장 이력 10건 조회
- 유형별 결과 비교 확인

## 6. 차별점

- 서로 다른 자산 유형을 하나의 인터페이스에서 평가한다.
- 수치만 보여주지 않고 설명 문구와 해석을 함께 제공한다.
- 입력 요약과 민감도 분석을 한 화면에서 확인할 수 있다.

## 7. 화면 구성

- 상단 소개 영역: 서비스 목적과 사용 흐름 안내
- 입력 영역: 주식, 채권, 프로젝트 평가 폼
- 결과 영역: 평가 결과 카드, 위험등급, 상세 보기, 저장 버튼
- 최근 이력 영역: 최근 저장 결과 목록

## 8. 기대 효과

- 평가 시간 단축
- 리스크 중심 의사결정 보조
- 시연과 내부 검토에 바로 사용할 수 있는 프로토타입 확보

## 9. 배포 운영 기준

- 프론트엔드: Vercel
- 백엔드: Render
- 데이터베이스: Supabase PostgreSQL
- 백엔드 API 주소: `https://ai-asset-evaluation-prototype.onrender.com`
- 프론트엔드 환경변수: `VITE_API_BASE_URL`
