# 개발문서

## 1. 기술 스택

- Frontend: React 18, Vite 5
- Backend: Spring Boot 3.5, Spring Web, Spring Data JPA, Validation
- Database: PostgreSQL
- Build/Deploy: Gradle, Docker, Render, Vercel

## 2. 시스템 구성

- 프론트엔드는 단일 페이지 대시보드에서 주식, 채권, 프로젝트 평가 폼을 제공한다.
- 백엔드는 `/api/evaluations` 하위 REST API로 평가 계산, 저장, 최근 이력 조회 기능을 제공한다.
- 평가 결과는 PostgreSQL의 `evaluation_records` 테이블에 저장된다.
- 프론트는 `VITE_API_BASE_URL` 환경변수로 백엔드 주소를 주입받는다.
- 로컬 개발 시 `VITE_API_BASE_URL`이 없으면 `/api` 프록시를 `localhost:8080`으로 연결한다.

## 3. 주요 구현 기능

### 3.1 주식 평가

- 입력값: 종목명, 현재 주가, 기대 연수익률, 변동성, 보유일수, 수량
- 산출값:
- 기대가치
- `VaR 95%`
- 종합점수
- 위험등급
- 계산 설명, 종합 해석, 하이라이트

계산 방식:
- 보유기간을 반영한 복리 기대수익 기반으로 기대가치 계산
- 파라메트릭 VaR(95%)로 핵심 위험지표 계산
- 기대수익률은 점수 가산, 변동성과 VaR 비율은 점수 감산

### 3.2 채권 평가

- 입력값: 채권명, 액면가, 쿠폰금리, 시장수익률, 만기
- 산출값:
- 현재가치
- 금리충격손실
- 수정 듀레이션
- 컨벡서티
- 종합점수
- 위험등급

계산 방식:
- 할인현금흐름으로 채권가치 계산
- 수정 듀레이션과 컨벡서티 계산
- `RATE_SHOCK = 1.25%` 기준 금리충격손실 추정
- 시장수익률, 듀레이션, 컨벡서티, 충격손실 비율을 반영해 점수 계산

### 3.3 프로젝트 평가

- 입력값: 프로젝트명, 초기 투자비, 할인율, 부도확률, 연도별 현금흐름
- 산출값:
- 현재가치
- NPV
- IRR
- 회수기간
- 신용위험추정치
- 종합점수
- 위험등급

계산 방식:
- 할인현금흐름 기반 NPV 계산
- 이분탐색으로 IRR 추정
- `Loss Given Default = 45%` 가정으로 신용위험추정치 계산
- 수익성 지표는 가산, 부도확률·회수기간·신용위험추정치는 감산

### 3.4 결과 저장 및 최근 이력 조회

- 사용자는 각 평가 결과를 `DB 저장` 버튼으로 저장할 수 있다.
- 저장 API는 프론트에서 전달한 평가 결과 객체를 받되, 서버에서 평가유형, 위험등급, 위험점수 범위, 상품별 허용 필드 조합을 1차 검증한 뒤 `evaluation_records`에 적재한다.
- 최근 이력 조회는 생성일시 역순 상위 10건을 반환한다.

## 4. 백엔드 구성

### 4.1 Controller

- `EvaluationController`
- `POST /api/evaluations/stock`
- `POST /api/evaluations/bond`
- `POST /api/evaluations/project`
- `POST /api/evaluations/save`
- `GET /api/evaluations/recent`
- `GET /api/evaluations/health`

예외 처리:
- 입력 검증 실패와 비즈니스 예외를 `400 Bad Request`로 응답한다.

### 4.2 Service

- `StockEvaluationService`: 기대가치, VaR, 종합점수 계산
- `BondEvaluationService`: 채권가치, 듀레이션, 컨벡서티, 금리충격손실 계산
- `ProjectEvaluationService`: NPV, IRR, 회수기간, 신용위험추정치 계산
- `RiskScoringService`: 점수 보정 및 등급 매핑
- `EvaluationPersistenceService`: DB 저장 및 최근 이력 조회

### 4.3 Entity

`evaluation_records`

- `id`
- `evaluation_type`
- `target_name`
- `present_value`
- `risk_metric_name`
- `risk_metric_value`
- `duration_value`
- `convexity`
- `npv`
- `irr`
- `payback_period`
- `risk_grade`
- `model_explanation`
- `commentary`
- `created_at`

## 5. 프론트엔드 구성

- 단일 `App.jsx` 중심 구성
- 주식/채권/프로젝트 입력 폼 3종
- 결과 카드, 상세 보기, 민감도 분석, 입력값 요약
- 프로젝트 평가의 연도별 현금흐름은 접기/펼치기 UI로 구성
- 채권의 쿠폰금리는 툴팁으로 설명 제공
- 최근 저장 이력을 유형별 컬럼으로 표시

- 자동 테스트 범위는 아직 제한적이며 저장 서비스 단위 테스트 중심으로만 보강되어 있다.
