# 개발문서

## 1. 기술 스택

- Frontend: React 18, Vite 5
- Backend: Spring Boot 3.5, Spring Web, Spring Data JPA, Validation
- Database: PostgreSQL (Supabase)
- Deploy: Render, Vercel
- Build: Gradle, Docker

## 2. 시스템 구성

- 프론트엔드는 Vercel에 배포된 React 단일 페이지 애플리케이션이다.
- 백엔드는 Render에 배포된 Spring Boot API 서버다.
- 데이터베이스는 Supabase PostgreSQL을 사용한다.
- 프론트엔드는 `VITE_API_BASE_URL` 환경변수로 백엔드 주소를 주입받는다.
- 백엔드는 `DB_URL`, `DB_USERNAME`, `DB_PASSWORD` 환경변수로 데이터베이스에 연결한다.

## 3. 주요 기능

### 3.1 주식 평가

- 입력값: 종목명, 현재 주가, 기대 연수익률, 변동성, 보유일수, 수량
- 출력값: 평가금액, VaR 95%, 종합점수, 위험등급, 설명 문구

### 3.2 채권 평가

- 입력값: 채권명, 액면가, 쿠폰금리, 시장수익률, 만기, 신뢰수준
- 출력값: 현재가치, 금리충격 손실추정, 듀레이션, 컨벡서티, 종합점수, 위험등급

### 3.3 프로젝트 평가

- 입력값: 프로젝트명, 초기 투자비, 할인율, 부도확률, 연도별 현금흐름
- 출력값: 현재가치, NPV, IRR, 회수기간, 신용위험추정치, 종합점수, 위험등급

### 3.4 저장 및 최근 이력 조회

- 사용자는 평가 결과를 DB에 저장할 수 있다.
- 백엔드는 최근 저장된 평가 결과 10건을 조회해 반환한다.
- 저장 대상 테이블은 `public.evaluation_records`다.

## 4. 백엔드 구성

### 4.1 API 엔드포인트

- `POST /api/evaluations/stock`
- `POST /api/evaluations/bond`
- `POST /api/evaluations/project`
- `POST /api/evaluations/save`
- `GET /api/evaluations/recent`
- `GET /api/evaluations/health`

### 4.2 주요 클래스

- `EvaluationController`: 평가 API 및 저장/조회 API 제공
- `StockEvaluationService`: 주식 평가 계산
- `BondEvaluationService`: 채권 평가 계산
- `ProjectEvaluationService`: 프로젝트 평가 계산
- `RiskScoringService`: 종합점수 및 위험등급 계산
- `EvaluationPersistenceService`: DB 저장 및 최근 이력 조회
- `EvaluationRecord`: 평가 결과 엔티티

### 4.3 데이터베이스 매핑

- 테이블명: `evaluation_records`
- JPA 설정: `spring.jpa.hibernate.ddl-auto=update`
- 애플리케이션 시작 시 엔티티 기준으로 테이블이 자동 생성 또는 업데이트된다.

주요 컬럼:

- `id`
- `evaluation_type`
- `target_name`
- `present_value`
- `risk_metric_name`
- `risk_metric_value`
- `duration_value`
- `convexity`
- `npv`
- `irr`
- `payback_period`
- `risk_grade`
- `model_explanation`
- `commentary`
- `created_at`

## 5. 프론트엔드 구성

- 단일 `App.jsx` 중심 구조
- 주식, 채권, 프로젝트 입력 폼 제공
- 결과 카드, 상세 보기, 민감도 분석, 최근 저장 이력 표시
- 저장 버튼 클릭 시 백엔드 `/save` API 호출

### 6.2 Supabase

- 데이터베이스: PostgreSQL
- 사용 테이블: `public.evaluation_records`
- 현재 구조에서는 Render 백엔드가 직접 DB에 접속한다.

### 6.3 Vercel

- Root Directory: `frontend`
- Framework Preset: `Vite`
- 주요 환경변수:
- `VITE_API_BASE_URL=https://ai-asset-evaluation-prototype.onrender.com`

## 7. 검증 항목

## 8. 현재 제약

- 자동 테스트 범위가 제한적이다.
- 인증/권한 관리 기능은 포함되어 있지 않다.
- 리포트 출력, 사용자 계정, 고급 필터링 기능은 아직 없다.
